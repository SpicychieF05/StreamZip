1. Product Overview

StreamZip is a web-based tool that allows users to download video or audio from a provided YouTube URL (single video or playlist) with strict limits and mobile-first optimization. The platform prioritizes fast, simple smartphone usage while remaining functional on desktop.

Primary Goal:
Provide a fast, clean, mobile-first experience for downloading permitted media content in MP4 or audio format.

Key Differentiator:

Mobile-first UX

Playlist ZIP support

Real-time job progress

Lightweight public MVP (<100 users/day)

2. Objectives & Success Metrics
Business Objectives

Launch stable MVP

Support public access

Maintain low infrastructure cost

Prevent abuse

5. Functional Requirements
5.1 URL Analysis

FR-1 User can paste a YouTube URL.
FR-2 System detects video vs playlist.
FR-3 System fetches metadata (title, thumbnail, duration).
FR-4 Invalid URLs show clear error.

5.2 Single Video Download

FR-5 User clicks "Download Video".
FR-6 System creates background job.
FR-7 Output format: MP4.
FR-8 Default quality: 720p.
FR-9 User sees progress indicator.
FR-10 User receives download link when ready.

5.3 Playlist Video Download

FR-11 Playlist videos are displayed in list.
FR-12 Each video has its own download button.
FR-13 “Download Playlist (ZIP)” available at bottom.
FR-14 Hard limit: max 8 videos for video ZIP.
FR-15 If playlist > 8, show warning and restrict.

5.4 Single Audio Download

FR-16 User clicks "Download Audio".
FR-17 System prefers M4A direct stream.
FR-18 If needed, system converts to MP3.
FR-19 User sees progress.

5.5 Playlist Audio ZIP

FR-20 “Download Audio Playlist (ZIP)” available.
FR-21 Hard limit: max 15 videos.
FR-22 Sequential processing in worker.
FR-23 ZIP streamed to user.

5.6 Job Management

FR-24 Each job has unique ID.
FR-25 User can see job status:

queued

processing

completed

failed

FR-26 Progress percentage shown when possible.
FR-27 Failed jobs show retry option.

5.7 Rate Limiting & Protection

FR-28 Per-IP download limit enforced.
FR-29 Playlist ZIP limited per hour.
FR-30 Bot protection enabled.
FR-31 Unsupported URLs blocked.

14. Risks & Mitigation
Technical Risk

yt-dlp breaks → keep auto-update plan

CPU spikes → queue limits

Disk overflow → auto cleanup

Platform Risk

Hosting restrictions → separate worker host

Abuse/bots → rate limiting + CAPTCHA

Product Risk

Slow playlist → sequential safe processing

Mobile UX issues → strict mobile-first testing
